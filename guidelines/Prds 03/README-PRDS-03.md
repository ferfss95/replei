# PRDs 03 — Índice

Documentação de produto derivada do **protótipo navegável** (dados fictícios). Cada PRD segue a estrutura: **Contexto → Usabilidade → Regras → Casos de Uso → Critérios de Aceite**.

| Arquivo | Escopo |
|---------|--------|
| `PRD-PLATAFORMA-FUNDACAO-ANALITICA.md` | Visão da plataforma, módulos, governança transversal |
| `PRD-EXPERIENCIA-WIZARD-JORNADA.md` | Jornada em etapas, navegação, estados |
| `PRD-DADOS-DIMENSOES-LOCAIS.md` | Território, rede, loja, canal |
| `PRD-DADOS-DIMENSOES-PRODUTO.md` | Sortimento, marca, franquia, hierarquias |
| `PRD-METRICAS-CATALOGO-SELECAO.md` | Catálogo, grupos, ordem de colunas |
| `PRD-TEMPORALIDADE-JANELAS-COMPARATIVOS.md` | Períodos, comparativos, baselines |
| `PRD-ANALISE-RESULTADOS-TABELA-GRAFICO.md` | Motor de resultado, gráfico, participação |
| `PRD-OPERACAO-EXPORTACAO-RESULTADOS.md` | Exportação e compartilhamento |
| `PRD-MODULO-PRODUTO-VENDA-ESTOQUE.md` | Lente Produto (visão mix + venda/estoque) |
| `PRD-MODULO-LOJA-PONTO-VENDA.md` | Lente Loja (território + produto embutido) |
| `PRD-MODULO-INDICADORES-KPI.md` | Lente Indicadores operacionais |
| `PRD-MODULO-EXTRAVIOS-ROADMAP.md` | Lente Extravios (estado atual + evolução) |

**Premissa:** o protótipo define **UX e fluxo**; produção exige **contratos de dados, segurança e NFR** ainda a formalizar onde indicado como decisão pendente.

**Padrão de nomenclatura:** `PRD-[DOMINIO]-[FEATURE].md` (domínio em maiúsculas, feature em kebab-case).

**Resumo para Excel:** `PRD-03-RESUMO-PARA-EXCEL.csv` (separador `;`, UTF-8 com BOM). No Excel: *Dados* → *De Texto/CSV* → codificação UTF-8 → delimitador ponto e vírgula.
